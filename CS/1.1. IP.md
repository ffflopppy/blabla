# IP 는 8비트 4개
IP 주소는 32 = 8비트씩 4옥텟(Octet)

- 11111111.11111111.11111111.11111111
- 255.255.255.255

# 서브넷 마스크
- 255.255.255.0/24
- 11111111.11111111.11111111.00000000/24
IP 주소에서 어디까지가 네트워크, 어디부터 호스트인지 구분하는값.
~~~
IP 주소:      192.168.1.1
서브넷 마스크: 255.255.255.0

IP 주소:      11000000.10101000.00000001.00000001
서브넷 마스크:  11111111.11111111.11111111.00000000

192.168.1  .  1
네트워크       호스트
~~~

같은 네트워크인지 판단할때 사용
- 서버A: 192.168.1.1 / 255.255.255.0
- 서버B: 192.168.1.2 / 255.255.255.0

네트워크 부분이 192.168.1 로 같으니까 같은 망 → 직접 통신 가능

- 서버A: 192.168.1.1 / 255.255.255.0
- 서버C: 192.168.2.1 / 255.255.255.0

네트워크 부분이 192.168.1 vs 192.168.2 다른 망 

# IP 클래스
IP를 사이즈별로 3개의 A/B/C 클래스로 나눔

## A
범위: 1.0.0.0 ~ 126.255.255.255
서브넷: /8

- 1옥텟이 네트워크 → 0~255 → 256개
  - 근데 실제로는 1~126만 사용 (0은 예약, 127은 루프백용으로 제외)
    
- 서브넷: 나머지 3옥텟 = 24비트
- 2²⁴ = 16,777,216개
- 256 × 256 × 256 = 16,777,216개


> 루프백 (Loopback) : 자기 자신에게 보내는주소(127.0.0.1) 내 PC → 127.0.0.1 → 다시 내 PC  
>>  0.0.0.0   → 모든 주소 또는 미지정  
>>  127.0.0.1 → 나 자신 (루프백)  
>>  255.255.255.255 → 브로드캐스트 (같은 망 전체)  
> 
> 서버가 0.0.0.0:8080 으로 열면  
> → 모든 네트워크 인터페이스에서 접속 받겠다는 뜻  
>
> 서버가 192.168.1.1:8080 으로 열면  
> → 그 IP로만 접속 받겠다는 뜻

> ## 네트워크 인터페이스  
> ### 물리 인터페이스
> ~~~
> eth0, eth1  → 유선 랜카드
> wlan0, wifi0 → 무선 랜카드
> ~~~
> ### 가상 인터페이스
> ~~~
> lo          → 루프백 (127.0.0.1)
> docker0     → Docker가 만든 가상 네트워크
> tun0        → VPN이 만든 가상 터널
> ~~~
> **서버에서 확인하는법**
> ~~~
> ip addr
> 1: lo → 127.0.0.1
> 2: eth0 → 192.168.1.1  (내부망)
> 3: eth1 → 10.0.0.1     (외부망)
>
> 실무에서
> eth0 → 내부망 (서버끼리 통신)
> eth1 → 외부망 (인터넷 통신)
> ~~~


# CIDR 표기
~~~
255.255.255.0 = /24   (1이 24개)
255.255.0.0   = /16   (1이 16개)
255.0.0.0     = /8    (1이 8개)
~~~
